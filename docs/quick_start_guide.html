V2N Robot Quick Start

Get your bowling pin navigation robot running in 5 minutes

Setup Run Controls Commands Examples Help

1 First Time Setup

Connect all hardware, then run one command to set everything up:

Hardware Connections

Arduino

/dev/ttyACM0

LiDAR

/dev/ttyUSB0

Camera

/dev/video0

Display

HDMI

Copy Package to V2N (From Your PC)

# Connect to V2N WiFi first, then run: $ scp -r bowling_target_nav root@192.168.50.1:~/ros2_ws/src/

Run Setup Script (On V2N)

$ cd ~/ros2_ws/src/bowling_target_nav/scripts $ ./v2n_master_setup.sh
✔
That's it! The setup script automatically:
  • Builds the ROS2 package
  • Installs autostart service
  • Starts the GUI
Your robot will now start automatically on every boot!

2 Running the Robot

Automatic Start (Default)

After setup, the robot starts automatically when powered on:

Power On → ~20 sec boot → GUI Ready → Press GO!

Manual Start / Stop

Command What It Does
./v2n_master_setup.sh --start Start the robot GUI
./v2n_master_setup.sh --stop Stop everything
./v2n_master_setup.sh --status Check if running
./v2n_master_setup.sh --build Rebuild the package

Using Systemd Service

$ systemctl status robot # Check status $ systemctl start robot # Start robot $ systemctl stop robot # Stop robot $ systemctl restart robot # Restart $ journalctl -u robot -f # View live logs

3 Controls

Keyboard Shortcuts

Key Action
G GO - Start navigating to detected bowling pin
Space STOP - Emergency stop (use anytime!)
Q or ESC Quit the application

GUI Layout

┌──────────────────────────────────────────────┐ │ │ │ ┌─────────────────┐ ┌─────────────────┐ │ │ │ │ │ │ │ │ │ SLAM Map │ │ Camera Feed │ │ │ │ (robot pos) │ │ (YOLO detects) │ │ │ │ │ │ │ │ │ └─────────────────┘ └─────────────────┘ │ │ │ │ [ GO Button ] [ STOP ] Status: Ready │ │ │ └──────────────────────────────────────────────┘
💡
Tip: The GO button only works when a bowling pin is detected. Look for the green bounding box in the camera feed!

4 Commands Reference

Master Setup Script

$ ./v2n_master_setup.sh [OPTION]
Option Description
--start Start the GUI (stops existing first)
--stop Stop all robot processes
--status Show running status
--build Build ROS2 package only
--service Install systemd autostart
--help Show help message

Test Runner Script

$ ./run_tests.sh [TEST_TYPE] [OPTIONS]
Test Type Description
arduino Test Arduino motor controller
lidar Test LiDAR sensor
camera Test camera + YOLO detection
all Run all tests (default)
--check Check hardware only
--gui Open interactive test GUI

Display Environment Variables

Variable Purpose Default
GDK_BACKEND GTK display backend wayland
WAYLAND_DISPLAY Wayland socket name wayland-0
XDG_RUNTIME_DIR Runtime directory /run

6 Quick Troubleshooting

Robot Not Moving?

# 1. Check Arduino $ ls -la /dev/ttyACM0 # 2. Restart robot $ ./v2n_master_setup.sh --stop $ ./v2n_master_setup.sh --start

No Camera Image?

# Check camera device $ ls -la /dev/video0 # Test camera $ ./run_tests.sh camera

GUI Not Showing?

# Set display $ export DISPLAY=:0 $ ./v2n_master_setup.sh --start
⚠
Still stuck? Run ./run_tests.sh --check to see which hardware is detected, then check logs with journalctl -u robot -f

What Is This Robot?

The V2N Bowling Target Navigation Robot is an autonomous mobile robot that can:

🔎

Detect Bowling Pins

Uses AI (YOLO) or V2N's DRP hardware to find bowling pins in camera images

🚗

Navigate Autonomously

Drives toward detected pins while avoiding obstacles using LiDAR

🗺

Build Maps

Creates 2D maps of the environment using SLAM (Simultaneous Localization and Mapping)

🎮

Easy Control

Simple GUI with GO/STOP buttons, or control remotely from your PC

In Simple Terms

Think of it like this: The camera is the robot's eyes (finds pins), the LiDAR is like sonar (avoids walls), and the Arduino controls the wheels. The brain (state machine) decides what to do based on what the sensors see.

Detection Process

Camera Frame (640x480 BGR) │ ▼ ┌─────────────────────────────────────┐ │ PREPROCESSING │ │ • Resize to 640x640 │ │ • Convert BGR → RGB │ │ • Normalize pixels (0-1) │ └─────────────────┬───────────────────┘ │ ▼ ┌─────────────────────────────────────┐ │ AI INFERENCE │ │ ┌─────────────┐ ┌─────────────┐ │ │ │ YOLO ONNX │ │ DRP BINARY │ │ ← Choose one │ │ (Software) │ │ (Hardware) │ │ │ └─────────────┘ └─────────────┘ │ └─────────────────┬───────────────────┘ │ ▼ ┌─────────────────────────────────────┐ │ POSTPROCESSING │ │ • Filter by confidence (>50%) │ │ • Apply NMS (remove duplicates) │ │ • Calculate center position │ └─────────────────┬───────────────────┘ │ ▼ Detection Result: • Class: "bowling_pin" • Confidence: 0.95 • Position: (320, 240) • Distance: ~1.5 meters

Distance Estimation

The robot estimates how far away the pin is based on how big it looks in the camera:

The Formula

distance = (real_height × focal_length) / pixel_height

  • real_height = Actual pin height (0.38 meters)
  • focal_length = Camera property (~500 pixels)
  • pixel_height = How tall the pin looks in the image

Example: If the pin appears 190 pixels tall → distance ≈ 1 meter

Movement Control

The robot uses mecanum wheels for omnidirectional movement:

FRONT ┌─────────────┐ │ ╲ ╱ │ │ FL FR │ FL = Front Left │ │ FR = Front Right │ RL RR │ RL = Rear Left │ ╱ ╲ │ RR = Rear Right └─────────────┘ REAR Movement combinations: • All forward → Robot moves forward • All backward → Robot moves backward • FL+RR forward, FR+RL backward → Robot moves LEFT • FR+RL forward, FL+RR backward → Robot moves RIGHT • Left side forward, right side backward → Robot ROTATES

Detection System

The robot supports multiple detection backends. You can switch between them without changing code!

Available Options

🤖

YOLO ONNX

Default

  • Works everywhere
  • Uses ONNX Runtime
  • CPU or GPU
  • ~100ms inference
⚡

DRP Binary

V2N Hardware

  • V2N accelerated
  • Uses DRP processor
  • External binary
  • ~50ms inference
🛠

Mock

Testing

  • No camera needed
  • Fake detections
  • For development
  • Instant

How to Switch Detection Backend

Method 1: Configuration File (Recommended)

Edit config/robot_config.yaml:

# Choose your detector type: # Options: "yolo_onnx", "drp_binary", "mock" detection: detector_type: "drp_binary" # ← Change this line confidence_threshold: 0.5 target_class: "bowling_pin" # DRP-specific settings (only used when detector_type is "drp_binary") drp_binary: binary_path: "/opt/drp/bowling_detector" input_method: "shared_memory" timeout: 1.0

Method 2: Environment Variable (Quick Override)

# Override detector type without editing files export V2N_DETECTOR_TYPE=drp_binary # Specify custom DRP binary path export V2N_DRP_BINARY=/path/to/your/detector # Or use YOLO with a different model export V2N_DETECTOR_TYPE=yolo_onnx export V2N_YOLO_MODEL=/path/to/custom_model.onnx # Now start the robot - it will use your settings ./v2n_master_setup.sh --start

Method 3: In Code (For Developers)

from bowling_target_nav.detectors import create_detector # Create YOLO detector detector = create_detector("yolo_onnx", model_path="models/bowling_yolov5.onnx" ) # Or create DRP detector detector = create_detector("drp_binary", binary_path="/opt/drp/bowling_detector" ) # Or create mock for testing detector = create_detector("mock", detection_mode="always" ) # Use it (same API for all detectors!) result = detector.detect(camera_frame) if result.has_detections: pin = result.best_detection print(f"Found pin at distance {pin.distance}m")

DRP Binary Protocol

If you're developing a custom DRP detector, here's the communication protocol:

INPUT (Your program reads this): ┌────────────────────────────────────────────┐ │ HEADER (16 bytes) │ │ ├─ width (4 bytes, uint32) = 640 │ │ ├─ height (4 bytes, uint32) = 480 │ │ ├─ channels (4 bytes, uint32) = 3 │ │ └─ format (4 bytes, uint32) = 0 (BGR) │ ├────────────────────────────────────────────┤ │ IMAGE DATA (width × height × channels) │ │ = 640 × 480 × 3 = 921,600 bytes │ │ Format: BGR, row-major order │ └────────────────────────────────────────────┘ OUTPUT (Your program writes this): ┌────────────────────────────────────────────┐ │ HEADER (8 bytes) │ │ ├─ num_detections (4 bytes, uint32) │ │ └─ status (4 bytes, uint32) = 0 │ ├────────────────────────────────────────────┤ │ DETECTION 0 (28 bytes) │ │ ├─ x1 (4 bytes, uint32) │ │ ├─ y1 (4 bytes, uint32) │ │ ├─ x2 (4 bytes, uint32) │ │ ├─ y2 (4 bytes, uint32) │ │ ├─ class_id (4 bytes, uint32) = 0 │ │ ├─ confidence (4 bytes, float) = 0.95 │ │ └─ reserved (4 bytes) │ ├────────────────────────────────────────────┤ │ DETECTION 1, 2, ... N │ └────────────────────────────────────────────┘

Navigation Logic

The robot's behavior is controlled by a state machine - a simple way to organize different modes of operation.

State Machine Diagram

START │ ▼ ┌───────────────┐ │ IDLE │ Robot is waiting │ (waiting) │ Press GO to start └───────┬───────┘ │ GO pressed ▼ ┌───────────────┐ Target lost │ DETECTING │ Looking for pins ┌────────────────►│ (scanning) │◄─────────────────┐ │ > 10 sec └───────┬───────┘ │ │ │ Pin found! │ │ ▼ │ │ ┌───────────────┐ │ │ │ NAVIGATING │ Moving toward pin │ │ (moving) │──────────────────┤ │ └───────┬───────┘ │ │ │ Obstacle! │ Obstacle cleared │ ▼ │ │ ┌───────────────┐ │ │ │ AVOIDING │ Going around │ │ │ (dodging) │──────────────────┘ │ └───────────────┘ │ │ ┌───────────────┐ └─►│ SEARCHING │ Rotating to find lost target │ (spinning) │ (gives up after 30 seconds) └───────────────┘ EMERGENCY STOP (Space key) → Can be triggered from ANY state

State Details

State What Happens Movement Transitions To
IDLE Robot waits for GO command Stopped DETECTING (on GO)
DETECTING Camera scans for bowling pins Stopped NAVIGATING (pin found)
SEARCHING (after 10s)
NAVIGATING Drives toward detected pin Forward + turning AVOIDING (obstacle)
DETECTING (pin lost)
SEARCHING Rotates to find lost pin Spinning in place NAVIGATING (pin found)
IDLE (timeout 30s)
AVOIDING Maneuvers around obstacle Turn away DETECTING (path clear)
STOPPED Emergency stop active Stopped IDLE (manual reset)

Navigation Parameters

These settings control how the robot moves:

navigation: # Speed limits max_linear_speed: 0.3 # Maximum forward speed (m/s) max_angular_speed: 0.5 # Maximum rotation speed (rad/s) # Default speeds default_linear_speed: 0.15 # Normal driving speed default_angular_speed: 0.3 # Normal turning speed # Obstacle avoidance obstacle_distance_threshold: 0.4 # Stop if obstacle closer than this (meters) obstacle_slowdown_distance: 0.8 # Start slowing down at this distance # Target approach target_reached_distance: 0.3 # "I'm there" when this close (meters) # Search behavior (when target is lost) search: enabled: true lost_timeout: 10.0 # Start searching after this many seconds search_timeout: 30.0 # Give up searching after this long angular_speed: 0.25 # How fast to spin while searching

Troubleshooting

Common Issues

Robot Not Moving

Symptoms: Press GO but nothing happens

Check:

  • Is Arduino connected? ls /dev/ttyACM0
  • Is a target detected? Check camera feed for green box
  • Check logs: journalctl -u robot -f

Fix: ./v2n_master_setup.sh --stop && ./v2n_master_setup.sh --start

No Detection (Camera Working)

Symptoms: Camera shows image but no detection boxes

Check:

  • Is the bowling pin in frame?
  • Is confidence threshold too high? Try 0.3
  • Is the YOLO model file present?

Fix: Check config/robot_config.yaml confidence_threshold

GUI Not Starting

Symptoms: Command runs but no window appears

Check:

  • Is display connected? echo $DISPLAY
  • Is Wayland/X11 running?

Fix: export DISPLAY=:0 then restart

Debug Commands
